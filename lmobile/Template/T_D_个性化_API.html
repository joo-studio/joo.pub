接口说明
接口地址
地址 描述
http://cf.51welink.com/Individuation/IndividuationService.asmx 个性短信
提交个性短信调用IndividuationSmsCommit方法
类型 命名 描述
接口定义 IndividuationSmsCommit(string User,string Password,string CorpId,string ProductId,string Key,string SmsVariable,string SmsTemplate) 提交短信
参数 User 提交账户
参数 Password 提交账户的密码
参数 Corpid 企业代码（扩展号，不确定请赋值空）
参数 ProductId 产品编号
参数 Key 用户自定义编号
参数 SmsVariable 短信变量
参数 SmsTemplate 短信模板
【短信变量】参数说明
结构说明：
<ISMV>  /*结构头标识*/
  <VU>
     <VT><V>13810252940</V></VT>  /*此出必须为手机号码，不能是其他内容*/
     <VT><V>P1</V></VT>     /*第一个参数名称，会作为信息内容显示*/
     <VT><V>P2</V></VT>     /*第二个参数名称，会作为信息内容显示*/
     <VT><V>P3</V></VT>     /*第三个参数名称，会作为信息内容显示*/
     <VT><V>PN</V></VT>     /*第N个参数名称，会作为信息内容显示，N，参数数量，根据实际需求定义*/
 </VU>
 <VU>
     <VT><V>13810252940</V></VT>  /*此出必须为手机号码，不能是其他内容*/

以上结构中：
 <VU>
     <VT><V>13810252940</V></VT>  /*此出必须为手机号码，不能是其他内容*/
     <VT><V>P1</V></VT>     /*第一个参数名称，会作为信息内容显示*/
     <VT><V>P2</V></VT>     /*第二个参数名称，会作为信息内容显示*/
     <VT><V>P3</V></VT>     /*第三个参数名称，会作为信息内容显示*/

作为一个号码单元，如果要发送至多个号码则需要多个这个的单元，上述的结构示例中表示信息要发送到2个手机号码上。
实际参数示例：
<ISMV><VU><VT><V>13810252940</V></VT><VT><V>A1</V></VT><VT><V>B2</V></VT><VT><V>C3</V></VT></VU><VU><VT><V>13810252940</V></VT><VT><V>A1</V></VT><VT><V>B2</V></VT><VT><V>C3</V></VT></VU></ISMV>
【短信模板】参数说明
结构说明：
{##}P1的内容{##}P2的内容{##}P3的内容……{##}PN的内容
其中【{##}】的个数要和短信变量参数中的参数的个数和顺序对应。
【{##}】相当于各参数的占位符
实际参数示例
{##}提交{##}个性{##}信息
返回值说明
返回值CSubmitState结构说明
命名 类型 描述
State int //提交结果值,返回值为0时，表示提交成功
MsgID string //短信批次号，其值为调用接口时传入的参数[key]的值。
MsgState string //审查是否通过标记
Reserve int //保留，扩展用
返回值枚举
State MsgState
0 提交成功
-99 异常
1006 变量文件格式不正确
1007 模版文件格式不正确 模板：{#变量名称#}
1008 提交队列失败
2000 提交参数不可为空

变量文件模板注意事项：每个VU代表一行变量 ，VU里面可以包含多个VT，每个VT中的V就是一个模版要替换的变量值但是每个VU的第一个VT的V必须是手机号码
