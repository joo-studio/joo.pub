接口说明
状态报告收取

如果客户需要接收状态报告，在相关的申请成功后，可以按照下面的说明进行编程接收
提供状态报告URL（例如：http://XXX.XXX.XXX.XXX/Up.asp） 以便我们给予绑定。

页面推送参数
类型 命名 描述
参数 AccountID 客户账号
参数 MsgID MessageID(短信提交成功后会生成MsgID)
参数 MobilePhone 客户提交手机号码
参数 ReportResultInfo 状态报告信息
参数 ReportState 状态报告，bit类型
参数 ReportTime 状态报告时间
参数 SendResultInfo 信息发送结果信息
参数 SendState 信息发送状态，bit类型
参数 SendedTime 信息发送时间
参数 SPNumber 长号码（下发端口号）


说明：
1、用户自行写进自己的数据库，以便接收数据。数据接收后请先缓存数据，否则将导致状态报告接收被降权
2、MsgID与提交返回的MsgID一致。SendState只有两个值（bit类型），False（数据库存储为0）或者True（数据库存储为1），其中False是发送失败，True是发送成功，ReportState只有两个值（bit类型），False（数据库存储为0）或者True（数据库存储为1），其中True是状态报告成功即成功送达到手机上，False是失败。ReportResultInfo是状态报告的具体信息，譬如状态报告失败的错误代码信息，如ReportResultInfo里包含DB:0102，那说明用户手机停机。
